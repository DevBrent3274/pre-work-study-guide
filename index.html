  • Put the meta data in your head and it is god practice to always have the basics that tell character set, screen sizes adjust and a title
  • The body element represents the visible content shown to the user.
  • The style element defines the CSS styles associated with a webpage
  • The base elemnet defines the base URL for a webpage
  • The link element connects an external resource to the HTML document
  • The meta element difines metadata such as the character set, description, keywords,author, and viewport
  • The body element defines the body of the document, which contains the content that is visible to the viewer. All the content we want to display to the user must be inside the body * body is a good example of a semantic HTML element where the name of the element describes its content and role it plays on a webpage - use them where possible to make your HTML easier to read!
  • heading and paragraph elements are pretty intuitive
  • The a element creates links
  • The img element is used for images
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap and navigation, or you can just make a statement
